# 1. 数据缩放与变换

## 1. Z-score (Standardization / 标准化)

- **公式**：$x' = \frac{x - \mu}{\sigma}$ （$\mu$ 为均值，$\sigma$ 为标准差）
    
- **解释**：将数据转化为均值为 0，方差为 1 的标准正态分布。它能消除量纲差异，让损失函数的等高线变圆。
    
- **适用场景**：对**距离或梯度敏感**的模型（逻辑回归、SVM、KNN、神经网络），且数据中没有极端的离群异常值。
    
- **面试 Q&A**：
    
    - **问题**：如果不做 Z-score，逻辑回归的梯度下降会遇到什么问题？
        
    - **答案**：如果特征尺度差异大，损失函数的等高线会变成极度扁平的椭圆。在窄的方向上梯度极大容易震荡，在宽的方向上梯度极小收敛缓慢，导致模型极难找到最优解。
        

## 2. Robust Scaler (稳健标准化)

- **公式**：$x' = \frac{x - \text{Median}}{\text{IQR}}$ （$\text{Median}$ 为中位数，$\text{IQR}$ 为四分位距）
    
- **解释**：利用中位数和四分位距进行缩放，不依赖容易被带偏的均值和标准差。
    
- **适用场景**：对距离/梯度敏感的模型，但**数据集中存在明显的极端异常值**（离群点）。
    
- **面试 Q&A**：
    
    - **问题**：为什么在有异常值的情况下，Robust Scaler 比 Z-score 更稳？
        
    - **答案**：Z-score 使用的“均值”对异常值极度敏感（一个超大值会拉高整体均值）。而 Robust Scaler 使用“中位数”，它只关注数据排序后的中间位置，能完美过滤掉两端极端异常值的干扰。
        

## 3. Min-Max (Normalization / 归一化)

- **公式**：$x' = \frac{x - \min(x)}{\max(x) - \min(x)}$
    
- **解释**：将所有数据强行按比例压缩到特定的边界内（通常是 $[0, 1]$）。
    
- **适用场景**：业务/UI 明确**需要取值边界**（如 0-100% 的打分），或者模型输入有严格范围要求（如图像像素 $0-255$ 缩放到 $0-1$ 喂给 CNN）。
    
- **面试 Q&A**：
    
    - **问题**：为什么在普通的机器学习回归/分类任务中，不首选 Min-Max？
        
    - **答案**：因为它对异常值“极度敏感”。如果混入一个超级大值，为了把它压缩到 1，其他所有正常数据都会被挤压成接近 0 的一团死水，导致模型无法区分正常数据的差异。
        

## 4. Log1p 变换 (平滑对数)

- **公式**：$y = \ln(x + 1)$
    
- **解释**：对数变换能把乘性关系转化为加性关系，有效压缩数据右侧的长尾。加 1 是为了防止 $x=0$ 时出现数学错误。
    
- **适用场景**：**正偏态/长尾分布**（如收入、点击量、购买频次），且数据包含 $0$ 甚至大量微小正数。
    
- **面试 Q&A**：
    
    - **问题**：为什么实际工程中常用 `log1p` 而不是直接用 `log`？
        
    - **答案**：因为真实业务数据（如用户的广告点击次数）常常包含 $0$。$\log(0)$ 是无定义的，会直接导致程序报错；而 $\text{log1p}(0) = \ln(1) = 0$，既保留了 0 的业务含义，又保证了计算安全。
        

## 5. Box-Cox 与 Yeo-Johnson (幂变换)

- **公式 (Box-Cox)**：$y = \frac{x^\lambda - 1}{\lambda}$ （当 $\lambda \neq 0$ 时）
    
- **解释**：通过寻找最优的参数 $\lambda$，尽可能将非正态的数据“掰”成正态分布。
    
- **适用场景**：模型假设数据服从正态分布，或者需要强力修正极度偏态的数据。
    
- **面试 Q&A**：
    
    - **问题**：Box-Cox 和 Yeo-Johnson 有什么核心区别？
        
    - **答案**：Box-Cox 的硬伤是**要求输入数据必须严格大于 0**。如果业务数据中包含 $0$ 或负数，必须使用它的升级版 Yeo-Johnson，它在底层设计了分段函数来兼容负数和 $0$。
        

## 6. 树模型 (Random Forest / XGBoost 等)

- **公式**：无（依赖信息增益/基尼系数进行分裂）
    
- **解释**：树模型寻找分裂点只看特征数值的**相对排序**，不看绝对大小。
    
- **适用场景**：各种表格类结构化数据的建模。
    
- **面试 Q&A**：
    
    - **问题**：如果我对特征做 Min-Max 归一化或者 Log 变换，会改变随机森林的训练结果吗？为什么？
        
    - **答案**：不会改变。树模型依靠数值的“排名”来切分节点（比如在第 5 大和第 6 大的值中间切一刀）。Min-Max 和 Log 都是“单调保序变换”，它们只改变数值大小，不改变数值前后的排名。因此，树模型分出的左右子树样本完全一致，信息熵的计算结果也完全一致。
        

## 7. 工程排雷：数据泄漏 (Data Leakage)

- **操作准则**：`fit` 只能在训练集做，`transform` 在训练集和测试集做。
    
- **解释**：任何缩放器的参数（均值、方差、最大最小值）都只能从训练集中学习，然后复用到验证集/线上数据中。
    
- **面试 Q&A**：
    
    - **问题**：把数据集拆分成 Train 和 Test 之前，我先对全量数据做一次统一的 Z-score，这样做对吗？
        
    - **答案**：绝对错误，这会导致**数据泄漏**。测试集在逻辑上代表“未来不可见的数据”。如果提前做全局 Z-score，测试集的信息（最大值、均值等）就会污染缩放参数，让模型在训练时“作弊”偷看到了未来的分布，导致线上真实效果严重翻车。
